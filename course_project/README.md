# Курсовой проект - база данных на основе сервиса Spotify

<b>Описание базы данных:</b>
<br>База данных представляет собой хранилище, основанное на интерфейсе сервиса Spotify.
<br>В базу данных входят:
* Таблицы (11шт)
* Представления (2шт)
* Процедуры (1шт)
* Триггеры(1шт)

<b>Таблицы:</b>
* users - пользователи сервиса с данными о них (имя, пол, дата рождения)
* tracks - треки, которые представлены на сервисе (название, жанр, ссылка на альбом)
* albums - альбомы, которые представлены на сервисе (название, дата выхода, жанры, ссылка на артиста)
* artists - артисты, которые представлены на сервисе (имя, страна происхождения)
* compilations - подборки, созданные сервисом (название, входящие в подборку треки)
* genres - словарь жанров музыки
* favourites - избранные треки пользователей
* dislikes - треки, которые не появляются в подборках из-за того, что пользователи поставили им дизлайк
* users_playlists - плейлисты, созданные пользователями (название, id создателя)
* users_playlists_subscriptions - таблица связей плейлистов и подписчиков
* users_playlists_tracks - треки, входящие в плейлисты


<b>Представления:</b>
* artist_album_song - объединенное представление с песнями, альбомами и артистами
* compilation_songnames - представление, в котором собраны подборки с названиями треков

<b>Процедуры:</b>
* time_rand_song - в зависимости от времени суток выдает случайную композицию по жанру заданному для данного отрезка времени суток


<b>Триггеры:</b>
* check_tracks_insert - выдает ошибку при попытке добавить новый трек с незаполненными полями name и album_id
* check_tracks_update - выдает ошибку при попытке обновить данные о треке с незаполненными полями name и album_id
